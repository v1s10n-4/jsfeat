Three Architectures to Replace the Morph Blob

Each keeps contour-based detection (proven best for cards) but replaces the bottleneck differently.

A

Multi-Scale Morph Cascade ★ RECOMMENDED

Run morph blob at 3 scales (radius 3, 5, 8). Use the smallest scale that detects a card. Small radius = tight blob = precise corners. Large radius = connected blob = catches hard cases.

Pros

  • Minimal code change — reuse existing pipeline 3x
  • Small radius fixes accuracy for "Detected but off" cases
  • Large radius fixes "No Card" cases (blob connectivity)
  • No new algorithms needed

Cons

  • 3x morph cost (but morph is only ~20ms)
  • Still fundamentally blob-based
  • Won't help border-touching cards
B

Edge Segment Grouping (Line-Based Detection)

Skip blobs entirely. Trace connected edge pixels into line segments, group by angle into 2 perpendicular directions, find 4 lines forming a rectangle, intersect for corners.

Pros

  • Works directly with edges — no blob noise
  • Corners come from line intersections — geometrically precise
  • Handles perspective better (lines adapt to card geometry)
  • Handles border-touching cards (lines don't need closed contours)

Cons

  • Significant new code (~300 lines) — segment tracing, line fitting, grouping, rectangle scoring
  • Wood grain produces many spurious segments
  • Rounded card corners break line segments
  • Needs robust RANSAC-like line fitting
C

Multi-Channel Contour Fusion

Run independent contour detection on grayscale, R-B warmth, and chroma channels. Each channel sees different card borders. Score and merge contour candidates across all channels.

Pros

  • Each channel may detect borders the others miss
  • Color channels already proven useful (Game 3 warmth breakthrough)
  • Naturally handles dark-on-dark and low-contrast edges

Cons

  • 3x full pipeline cost (3 Canny + 3 morph + 3 findContours)
  • Still blob-based per channel — same precision ceiling
  • Merging contour candidates from different channels is tricky

Hybrid Possibility: A + B

Cascade: Multi-Scale Morph → Line-Based Fallback

Use A as the primary detector (fast, proven). When it fails, fall back to B (line segment grouping) for the hardest cases. This gives the best of both: speed for easy cases, geometric precision for hard cases.
